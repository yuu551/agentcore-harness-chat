# WAF による IP 制限

IP 制限のやり方は 2 つあります。守る対象とコストが違うので、目的に合わせて**どちらか 1 つ**を選んでください。

## どちらを使うか（選択ガイド）

| | A. テンプレートの WAF（このドキュメント） | B. Amplify コンソールの Firewall |
|---|---|---|
| 守る対象 | **ログイン操作**（Cognito への認証通信） | **画面の配信**（ページ自体を見せない） |
| 許可 IP 外からの見え方 | ログイン画面は見えるが、サインインが 403 で失敗 | ページ自体が開けない |
| 認証情報が漏れた場合 | 許可 IP 外からはログイン不可（**効く**） | Cognito・API は開いたままなので直接認証されうる（効かない） |
| 設定方法 | 環境変数 `ALLOWED_IPV4_CIDRS`（IaC 管理） | コンソールで設定（Hosting → Firewall → IP address protection） |
| 費用目安 | 約 $6〜7/月 | 約 $23/月（WAF 約 $8 + Amplify Firewall 利用料 $15） |
| 向いている場合 | アクセス制御として IP を縛りたい（推奨の既定） | ツールの存在自体を外部から隠したい |

- 迷ったら **A（テンプレートの WAF）** を推奨します。実害のある操作（認証・API）を縛るのはこちらで、費用も安く、設定がコードとして残ります
- 両方を併用すると「完全に閉じた」状態にできますが、WAF のスコープ（REGIONAL / CLOUDFRONT）が異なるため **Web ACL は必ず 2 つ**になり、費用と許可リストの管理も二重になります。コンプライアンス要件がある場合のみ検討してください

以下は A（テンプレートの WAF）の設定手順です。B は末尾の [Amplify Hosting にも IP 制限をかける](#amplify-hosting-にも-ip-制限をかけるオプション) を参照してください。

## 設定方法

許可する CIDR をカンマ区切りで指定してデプロイします。IPv4 / IPv6 は片方だけでも構いません。**どちらも未指定の場合、WAF リソースは作成されません。**

```bash
ALLOWED_IPV4_CIDRS=203.0.113.0/24,198.51.100.10/32 \
ALLOWED_IPV6_CIDRS=2001:db8::/32 \
HARNESS_ARN=arn:... npx ampx sandbox --once
```

マスクなしの単一 IP（例: `203.0.113.10`）を指定した場合は、自動で `/32`（IPv6 は `/128`）を補完します。

自分のグローバル IP は次のコマンドで確認できます。

```bash
curl https://checkip.amazonaws.com
```

> **注意**: iCloud Private Relay や VPN を経由していると、ブラウザの外向き IP が上記コマンドの結果と異なる・接続のたびに変わることがあります。許可したはずの IP で弾かれる場合は、ブラウザ側の IP 確認サイトの結果と見比べてください。

## 作成されるリソースと動作

| リソース | 内容 |
|---|---|
| WAF IP Set（IPv4 / IPv6） | 許可 CIDR のリスト |
| WAF Web ACL（REGIONAL） | デフォルトアクション **block**、IP Set に一致するリクエストのみ allow |
| Web ACL Association | Cognito User Pool への関連付け |

許可 CIDR 外からアクセスすると、Cognito の認証エンドポイント（サインイン・トークン発行・サインアップなど）への通信がブロックされ、ログインできなくなります。ブロック状況は CloudWatch メトリクス（`harness-chat_cognito_waf`）で確認できます。

## 保護範囲（この機能を有効にしたときの状態）

| 対象 | 許可 IP 外からのアクセス |
|---|---|
| ログイン画面（Amplify Hosting の配信） | **見える**（この WAF の対象外） |
| ログイン操作（パスワード / SSO、Cognito への通信） | **ブロックされる**（403） |
| チャット API（API Gateway `/invoke`） | WAF 対象外だが、Cognito の JWT がないと呼び出せない |
| ログイン済みユーザーの継続利用 | **トークン失効（既定 1 時間）までは利用できる**。失効後はトークン更新も Cognito 経由のためブロックされる |

つまり「画面は見えるがログインできない = 実害のある操作はすべて認証の内側」という守り方です。静的アセットに秘密情報は含まれないため（User Pool ID や API URL は公開前提の設定値）、多くの社内利用ではこの構成で十分です。

## Amplify Hosting にも IP 制限をかける（オプション）

「許可 IP 外からはログイン画面の存在すら見せない」ことが要件になる場合（顧客のコンプライアンス要件など）は、Amplify Hosting の Firewall（WAF 統合）を追加します。Hosting アプリはこのテンプレートの IaC 管理外のため、コンソールでの設定になります。

1. Amplify コンソールで対象アプリを開き、「Firewall」を選択
2. 「IP address restriction」で許可する CIDR を登録して保存

> Amplify Firewall は CloudFront スコープの WAF が別途作成されるため、テンプレートの WAF とは統合できません。費用は WAF 約 $8/月 + Amplify Firewall 利用料 $15/月（コンソールの見積もり表示より）。冒頭の[選択ガイド](#どちらを使うか選択ガイド)のとおり、明確な要件がなければテンプレート側の WAF だけで運用を始めるのがおすすめです。

## コスト

WAF は有効化すると固定費が発生します（2026 年時点の目安）。

- Web ACL: $5.00/月
- ルール: $1.00/月 × ルール数（IPv4 + IPv6 で最大 2 ルール）
- リクエスト: $0.60/100 万リクエスト

ログイン時のみ通過するトラフィックのため、リクエスト課金は実質無視できる規模です。
